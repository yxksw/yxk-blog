要在真机上运行但不使用 Expo Go，本质上有两条路：**开发版（Dev Client）** 或 **直接生成原生工程运行（prebuild/run:android）**。前者更“Expo 风格”，也最常用。

**方案 A（推荐）：Dev Client + 真机热更新调试**

- 作用：在真机里装一个“你自己的调试 App（dev build）”，以后开发时用它打开项目（不是 Expo Go）。
- 步骤：
  - 安装 dev client 依赖：

  ```bash
    cd F:\ALL\React\New\NagoMusic
    npx expo install expo-dev-client
  ```

  - 构建并安装到真机（两种方式选其一）：
  - 用 EAS 云构建（最省事，但需要登录 Expo 账号）：

    ```bash
      npm i -g eas-cli
      eas login
      eas build:configure
      eas build --profile development --platform android
    ```

    构建完成后下载 apk 并安装到手机。

  - 本地跑原生构建（需要装 Android Studio/SDK/Java）：

    ```bash
      npx expo prebuild --platform android
      npx expo run:android --device
    ```

  - 开发时启动（让 dev client 连接你的 Metro）：

  ```bash
    npx expo start --dev-client
  ```

  然后在真机上打开你安装的 dev build，按提示连接（局域网/USB/隧道）。

**方案 B：直接真机安装“预览/生产包”（不热更新调试）**

- 作用：更像真正发布的安装包，适合给别人测试或自测安装。
- EAS：

  ```bash
  eas build --platform android --profile preview
  # 或 production
  eas build --platform android --profile production
  ```

**真机连接要点（Android）**

- 打开开发者选项 + USB 调试，数据线连接电脑。
- 确保电脑能识别设备（`adb devices` 里能看到设备）。
- 如果同一 Wi‑Fi 下局域网不通，可用 USB 或使用 `expo start --tunnel`（连接更稳但可能更慢）。
